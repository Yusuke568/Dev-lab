# 勤務報告書テンプレート化システム ドキュメント

## 📋 ドキュメント概要

本プロジェクトは、既存の勤怠管理システムをモダンなアーキテクチャへ刷新し、勤務報告書のテンプレート化、スマートフォン対応、および保守性の向上を実現したものです。

## 🚀 システムの進化 (過去 vs 現在)

本システムは、リファクタリングを通じて以下の3段階の進化を遂げました。

| 特徴 | レガシー期 (初期) | 移行期 (Layered MVC) | 現在 (Hexagonal/Clean) |
|------|-------------------|----------------------|-----------------------|
| **アーキテクチャ** | Servlet/JSP Model 2 | Layered MVC + FrontController | **Hexagonal Architecture** |
| **責務分離** | Servletにロジックが混在 | Service/DAO層の分離 | **Domain/UseCase/Adapterの完全分離** |
| **DBアクセス** | 手動JDBC管理 | 共通Baseクラスによる管理 | **Persistence Adapterを通じた抽象化** |
| **画面表示** | スクリプトレット (`<% %>`) | JSTL / EL式 | **JSTL / EL + 非同期API (JSON)** |
| **テスト容易性** | 困難 | 普通 (Service単位) | **極めて高い (Domain/UseCase単位でMock可能)** |
