# アーキテクチャ設計書 (最終版)

## 1. はじめに

### 1.1. 目的
本ドキュメントは、本システムの技術基盤である**ヘキサゴナルアーキテクチャ（ポート＆アダプタ）**の詳細を定義します。本システムは、レガシーなLayered MVCから、ドメイン駆動設計の原則を取り入れたモダンなアーキテクチャへと完全に移行されました。

### 1.2. 設計思想
- **ドメイン中心**: ビジネスロジックを「内側（Domain/Application）」に閉じ込め、技術的詳細を「外側（Adapter）」に分離。
- **依存関係の逆転 (DIP)**: ビジネスルールが外部技術（DB, Web等）に依存しないよう、インターフェース（Port）を介して接続。
- **高保守性・高テスト容易性**: 各コンポーネントが疎結合であり、ドメイン単位での単体テストが可能。

---

## 2. システム構造 (統一されたヘキサゴナル構造)

全てのコンポーネントは、以下の標準化されたパッケージ構造に従って配置されています。

```
com.example.<context>/
├── domain/
│   ├── model/         # 豊かなドメインモデル (Entity, Value Object)
│   └── port/
│       └── out/       # 外向きポート (Repository IF)
├── application/
│   ├── port/
│   │   └── in/        # 内向きポート (UseCase IF, Command)
│   └── service/       # ユースケース実装 (Transaction境界)
└── adapter/
    ├── in/
    │   └── web/       # 内向きアダプタ (Action, DTO, FrontController)
    └── out/
        └── persistence/ # 外向きアダプタ (PersistenceAdapter, SQL)
```

### 2.1. 実装済みコンテキスト
- **com.example.shain**: 社員管理（登録・更新・削除・一覧）
- **com.example.leave**: 有給休暇管理（自動付与・一括付与・日数調整）
- **com.example.auth**: 認証（ログイン・ログアウト）
- **com.example.kintai**: 勤怠管理（表示・更新）
- **com.example.adapter**: グローバルなインフラストラクチャ（FrontController, TransactionManager等）

---

## 3. 主要な設計要素

### 3.1. 依存性注入 (DI)
`com.example.kintai.configuration.DependencyFactory` が、全てのコンポーネントのインスタンス化と依存関係の解決を一元管理しています（手動DI）。

### 3.2. トランザクション管理
`com.example.application.port.out.TransactionManager` を通じて、サービス層で宣言的にトランザクションの境界を制御します。ThreadLocal を利用したコネクション管理により、アダプタ側でトランザクションを意識することなく一貫性を保ちます。

### 3.3. DTO (Data Transfer Object)
アダプタ（特にWeb）とアプリケーション層の間でドメインモデルを直接晒さず、JSPやAPIが期待する形式に変換したDTOを使用します。これにより、フロントエンドとバックエンドの変更を個別に管理可能です。

---

## 4. 開発ガイドライン

- **新機能の追加**: 既存のコンテキストに従うか、新しいコンテキストを作成し、ヘキサゴナル構造を維持してください。
- **ドメインの純粋性**: `domain` パッケージには、Java標準ライブラリ以外の技術的依存関係を持ち込まないでください。
- **テストの義務化**: 新しいドメインモデルやユースケースを追加する際は、必ず対応するテストコードを記述してください。
